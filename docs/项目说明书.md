# 项目说明书

## 1. 项目名称

《热龄卫士：基于多源时空数据的城市适老化热健康风险预警与避险服务调度平台》

## 2. 项目目标

面向武汉主城区，识别高温过程下老年人热健康高风险区域，评估现有避险资源的步行可达性，并在新增点位受限条件下给出临时纳凉点的最优布局建议。

## 3. 核心能力

1. 基于 `Open-Meteo` 获取未来温度与体感温度趋势
2. 基于 `OpenStreetMap / Overpass` 获取避险相关 POI
3. 基于 `WorldPop` 最新可用中国老年人口栅格构建老年人口空间分布
4. 基于 `Geofabrik` 路网数据构建真实步行网络
5. 生成网格级热健康风险与街区级风险排行
6. 输出 `3 / 5 / 8` 个新增点位情景下的优化结果
7. 自动监测武汉官方纳凉通报并将可定位在运点位纳入基线
8. 通过 `FastAPI + ECharts` 提供前后端展示与证据链说明

## 4. 技术架构

- 前端：原生 HTML / CSS / JavaScript + `ECharts`
- 后端：`FastAPI`
- 数据处理：`Python`
- 空间分析：`GeoPandas`、`NetworkX`
- 优化求解：`PuLP`

## 5. 数据来源

- 天气：`Open-Meteo`
- POI：`OpenStreetMap / Overpass API`
- 老年人口：`WorldPop` 最新可用中国年龄结构栅格
- 路网：`Geofabrik Hubei roads shapefile`
- 官方纳凉点：`武汉市政府 / 武汉市民政局 / 武汉市国防动员办公室` 公开页面

## 6. 目录说明

- `backend/`：后端接口
- `frontend/`：前端页面
- `scripts/`：数据抓取、处理、分析、导出脚本
- `data/processed/`：处理后的业务数据
- `outputs/report_tables/`：报告表格
- `outputs/report_charts/`：报告图表
- `docs/`：项目文档

## 7. 关键输出

- 风险摘要：`data/processed/risk_summary.json`
- 可达性摘要：`data/processed/accessibility_summary.json`
- 选址实验：`data/processed/optimization_experiments.json`
- 默认推荐点位：`data/processed/site_recommendations.json`
- 研究报告初稿：`docs/研究报告初稿-热龄卫士.md`

## 8. 当前实现特点

当前版本采用“覆盖优先 + 时间优化补位”的混合策略：先最大化 15 分钟阈值内可新增覆盖的高风险老年人口，再利用剩余点位继续缩短平均到达时间。

同时，项目首页提供“证据链”面板，显示上游数据源刷新状态、官方原文入口与方法参考链接，用于答辩时直接说明结果来源和可信性。

## 9. 适用场景

- 街道办高温应对
- 社区临时纳凉点布设
- 卫健与民政协同调度
- 比赛演示与研究报告支撑
