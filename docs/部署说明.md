# 部署说明

## 1. 部署位置

项目目录位于 `D:\bisai`，数据、虚拟环境、导出结果均保存在 `D:`。

## 2. 环境要求

- Windows PowerShell
- Python 3.11+
- 建议可联网以更新天气与 POI 数据

## 3. 首次部署

```powershell
cd D:\bisai
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python scripts\run_pipeline.py
```

## 4. 启动服务

### 方式一：一键启动

```powershell
cd D:\bisai
.\启动项目.ps1
```

### 方式二：手动启动

```powershell
cd D:\bisai
.\.venv\Scripts\Activate.ps1
uvicorn backend.app.main:app --host 127.0.0.1 --port 8000
```

浏览器访问：

- `http://127.0.0.1:8000/`

## 5. 更新数据

```powershell
cd D:\bisai
.\更新数据.ps1
```

推荐直接使用上面的脚本。当前脚本会执行完整真实数据流水线，自动刷新：

- `WorldPop` 最新可用中国老年人口栅格
- `Geofabrik` 湖北路网
- `Open-Meteo` 预报与历史热浪窗口
- 武汉官方纳凉通报与可定位官方点位

如需手动执行，也应直接运行主流水线，而不是逐个跑旧脚本：

```powershell
.\.venv\Scripts\python scripts\run_pipeline.py
```

## 6. 导出材料

```powershell
cd D:\bisai
.\生成材料.ps1
```

导出目录：

- 图表：`outputs/report_charts/`
- 表格：`outputs/report_tables/`
- 报告：`docs/研究报告初稿-热龄卫士.md`
- 参赛提交目录：`outputs/submission_package/待填作品编号-参赛总文件夹/`

## 7. 常见说明

1. 路网缓存文件在 `data/raw/walk_network.pkl`
2. 路网状态文件在 `data/raw/walk_network_status.json`
3. 当前已接入真实 `WorldPop` 与真实步行网络
4. 若需重建路网缓存，可删除上述两个路网文件后重新执行 `.\更新数据.ps1`
5. 网站首页“证据链”面板会显示本轮启动后各上游数据源的刷新状态与检查时间
6. 如需程序化查看上游刷新状态，可访问 `GET /api/data-sources`
7. `.\生成材料.ps1` 会同时生成图表/表格/报告初稿，以及按赛规整理好的参赛提交目录
