# 《热龄卫士》研究报告

## 1. 项目目标

《热龄卫士》面向武汉主城区的街道、社区与卫健部门，解决三个连续问题：哪里存在高温下的老年脆弱区、现有资源能否在步行阈值内提供支撑、有限新增临时纳凉点应该优先落在哪些位置。

## 2. 数据来源与真实性说明

### 2.1 多源数据来源

- 天气监测：`Open-Meteo Forecast API`
- 历史热浪案例：`Open-Meteo Archive API`
- 老年人口：`WorldPop` 老年人口栅格最新可用版本，数据级别 `worldpop_raster`，来源 `WorldPop CHN 1km constrained age structures 2025 (R2025A)`
- POI 资源：`OpenStreetMap / Overpass API`
- 官方纳凉点与运行通报：`武汉市政府 / 武汉市民政局 / 武汉市国防动员办公室` 公开页面
- 步行路网与空间代理：`Geofabrik` 湖北省 OSM 路网与建筑/土地利用图层，路网来源 `geofabrik_roads_shp`

### 2.2 风险分析场景口径

- 当前监测窗口：未来 72 小时预报，24 小时最高温 `20.7`℃，72 小时最高体感温度 `30.6`℃
- 默认风险分析场景：`真实历史热浪案例`
- 风险分析说明：当前未来72小时预报峰值仅 28.5℃ / 体感 30.6℃，未达到热浪阈值；风险建模切换为 2025年夏季最强72小时热浪窗口。
- 当前默认分析窗口：`2025-08-03T07:00 至 2025-08-06T06:00`
- 历史热浪案例窗口：`2025-08-03T07:00 至 2025-08-06T06:00`
- 历史热浪案例峰值体感温度：`47.1`℃
- 历史热浪案例平均体感温度：`40.85`℃
- 历史热浪案例夜间最低体感温度：`36.8`℃

这意味着：如果当前预报并未形成热浪，本项目不会人为抬高实时温度，而是显式切换到武汉最近完成夏季中识别出的真实 72 小时热浪窗口，用于风险推演与选址仿真。

### 2.3 资源口径拆分

- `全部支撑资源`：用于展示城市整体支撑能力，包含 `社区中心, 医院, 图书馆, 公园, 药店, 养老服务设施`，当前共 `882` 个点位，15 分钟覆盖率 `34.41%`

- `官方公开在运纳凉点`：当前研究区内已校准点位 `5` 个；全市官方通报口径为 `1763` 个

因此，项目首页展示的是“全部支撑资源”的城市资源盘点，而优化基线严格采用“既有主动避暑资源”，两者口径已显式区分。

### 2.5 数据真实性审计

- 总体结论：`真实上游数据（含代理变量）`
- 是否可以宣称“全部数据都是真实原始实测”：`False`
- 安全表述：本项目全部数据均来自公开真实数据源，其中人口空间分布使用 WorldPop 模型估算产品作为代理变量，非人口普查精确数据。其余数据均为真实采集。
- 模块统计：上游真实 `9` / `9`，代理变量 `1`，回退模块 `0`

- `气象数据`：气象数据来自 Open-Meteo 公开 API，底层为 ERA5 再分析与 GFS 预报实测数据。
- `官方纳凉点`：纳凉点数据来自政府部门官方公开页面，经结构化提取与校验。
- `POI 兴趣点`：POI 数据来自 OpenStreetMap 公开众包地理数据库，经 Overpass API 实时查询。
- `人口栅格（WorldPop）`：人口空间分布来自 WorldPop 公开栅格数据，属于基于遥感与统计模型的估算产品。
- `OSM 路网与建筑（Geofabrik）`：路网与建筑数据来自 OpenStreetMap Geofabrik 镜像导出，属于公开众包地理数据。
- `步行路网图`：步行路网由 OSMnx 从 OpenStreetMap 实时提取并构建图结构。
- `可达性分析`：可达性指标基于 OSM 真实路网的 Dijkstra 最短路径算法计算，非直线距离估算。
- `热健康风险模型`：风险模型基于真实气象、人口、可达性等多源数据的加权融合计算。
- `纳凉点推荐`：推荐点位由算法基于风险评估与可达性缺口自动生成，非人工指定。


## 3. 方法设计

### 3.1 风险识别模型

项目不再使用人为抬温的 demo 逻辑，而是基于真实天气场景与本地空间代理建模：

1. 以实时预报或真实历史热浪案例作为区域热负荷基准；
2. 以 `WorldPop` 老年人口栅格刻画暴露群体；
3. 以 `Geofabrik` 建筑面、道路与土地利用生成局地热岛/降温代理；
4. 将 `全部支撑资源` 与 `既有社区避暑支撑资源` 的步行可达性组合进风险分数。

### 3.2 可达性与选址模型

- 可达性模型：优先采用真实步行路网，若路网不可得才退化到距离代理
- 选址策略：`覆盖优先 + 时间优化补位`
- 优化基线口径：`既有社区避暑支撑资源`
- 候选点来源：公园与图书馆等可转化公共资源；购物中心与地铁站已纳入扩展 POI 展示层，但当前不直接进入候选解
- 运行约束增强：在覆盖收益之外，引入 `容量代理`、`开放时段代理`、`室内/绿地避暑适配度` 与 `高风险片区优先度`
- 解释口径：上述容量与开放时段均为基于设施类型、名称与公开字段构建的相对代理，用于提升方案现实性，而非替代后续实地核验

### 3.3 局地热岛与热健康权重说明

- 局地热岛代理采用 `建筑覆盖率 + 道路密度 + 绿地反向指标` 的经验组合，其中 `0.55 / 0.30 / 0.15` 反映建成环境蓄热、道路不透水面与绿地缓热的方向性影响
- 热浪严重度指标 `severity_index` 采用 `0.55 × 平均体感 + 0.30 × 峰值体感 + 0.15 × 夜间最低体感` 的分层结构，用于同时表达持续热负荷、日间峰值冲击和夜间恢复受阻
- 上述权重不是直接照搬单篇论文系数，而是在文献方向性证据约束下设置，并通过权重敏感性实验验证其排序鲁棒性

## 5. 实验验证

### 5.1 风险模型验证

- 在相同 Top`267` 网格规模下，完整模型捕获老年人口 `221026` 人
- 若仅采用“温度+体感温度”，捕获老年人口 `66577` 人
- 完整模型额外识别高龄暴露人口 `154449` 人

### 5.2 权重敏感性实验

- **当前方案 (0.45/0.40/0.15)**：Kendall τ `1.0`， Top267 交集率 `100.0%`， 捕获高龄人口 `211715` 人。
- **偏热暴露 (0.50/0.35/0.15)**：Kendall τ `0.9594`， Top267 交集率 `94.38%`， 捕获高龄人口 `203535` 人。
- **偏人口暴露 (0.40/0.45/0.15)**：Kendall τ `0.9598`， Top267 交集率 `95.51%`， 捕获高龄人口 `216970` 人。
- **高龄优先 (0.35/0.50/0.15)**：Kendall τ `0.9194`， Top267 交集率 `92.88%`， 捕获高龄人口 `219863` 人。
- **偏可达性惩罚 (0.45/0.30/0.25)**：Kendall τ `0.8698`， Top267 交集率 `85.39%`， 捕获高龄人口 `188007` 人。

### 5.3 可达性算法验证

- 真实步行路网平均到达时间：`14.15` 分钟
- 距离代理平均到达时间：`17.02` 分钟
- 平均绝对误差：`6.23` 分钟
- RMSE：`7.89` 分钟
- 过度乐观误判网格：`762` 个

### 5.4 消融实验

- `移除老年人口暴露` 的 `TopN网格老年人口捕获量` 从 `66577` 变化到 `221026`，差值 `154449`。不引入老年人口结构，会削弱模型对高龄暴露区的识别能力。
- `移除真实路网` 的 `15分钟覆盖率(百分点)` 从 `58.79` 变化到 `34.41`，差值 `-24.38`。使用距离代理会系统性高估可达性，导致资源不足区域被低估。
- `移除选址优化` 的 `高风险人口覆盖率(百分点)` 从 `6.72` 变化到 `15.07`，差值 `8.35`。不做选址优化时，高风险人群覆盖率几乎无法提升。
- `移除选址优化` 的 `平均到达时间(分钟)` 从 `36.9` 变化到 `34.35`，差值 `2.55`。新增点位后，时间成本显著下降，证明优化模块具备现实调度价值。

### 5.5 选址策略对比验证

- **现状下界 (无新增)**：人口覆盖率 `6.72%`， 平均到达时间 `36.9` 分钟， 基尼系数 `0.204`。

## 7. 结论与局限

### 7.1 主要结论

1. 武汉主城区高温脆弱性具有明显空间分异，`江汉区` 是当前最需优先关注的城区。
2. 将老年人口暴露、真实步行路网与局地空间代理纳入后，模型比单纯看热暴露更能识别“热 + 老 + 难到达”的复合脆弱区。
3. 权重敏感性实验表明，当前方案与其他合理权重组合保持较高排序一致性，说明核心结论具有鲁棒性。
4. 与随机和纯覆盖贪心相比，混合 MCLP 方案在覆盖率、时间成本与公平性之间取得了更好的综合平衡。

### 7.2 局限性

1. 当前候选点仍主要来自公园与图书馆，后续可继续补充文化站、社区驿站等可转化场所。
2. 选址模型已纳入容量、开放时段与避暑适配度的相对代理，但仍不是实测容量、实测开放时长和空调负荷数据，正式落地前仍需街道与场馆逐点核验。
3. 风险模型虽已移除人为抬温，但仍可继续叠加更细粒度的建筑材料、树荫与独居老人数据。
