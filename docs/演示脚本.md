# 演示脚本

## 1. 开场

“本项目叫热龄卫士，面向武汉主城区高温场景，解决三个实际问题：哪些老年人最危险、现有避险资源够不够、有限新增点位应该放在哪里最有效。” 

## 2. 展示风险识别

1. 打开首页
2. 说明当前温度、24 小时最高温、高风险网格数量
3. 展示风险热力矩阵与街区风险排行
4. 强调高风险并不是只看气温，而是综合了老年人口暴露与资源可达性

## 3. 展示可达性诊断

1. 展示现有避险资源数量
2. 说明 5/10/15 分钟覆盖率
3. 强调当前使用真实步行网络，而不是简单直线距离

## 4. 展示选址优化

1. 切到“新增点位情景对比”
2. 说明系统先最大化 15 分钟新增覆盖，再用剩余点位继续缩短平均到达时间
3. 对比基线、3 点、5 点、8 点方案
4. 说明当前共有 11 个高风险网格，其中 3 个能在 15 分钟阈值内通过候选点新增覆盖，因此 5 点和 8 点方案更多体现在平均到达时间继续下降

## 5. 展示推荐点位

1. 展示默认 5 点方案推荐表
2. 说明每个推荐点位包含新增覆盖人口、新增覆盖网格、改善网格、加权时间改善
3. 点出重点推荐点位，如常青藤口袋公园、菱角湖公园

## 6. 展示导出材料

1. 打开 `outputs/report_tables/选址优化情景对比.csv`
2. 打开 `outputs/report_charts/02_平均到达时间对比.png`
3. 打开 `docs/研究报告初稿-热龄卫士.md`

## 7. 结束语

“热龄卫士不是一个普通的大屏，而是把热风险识别、步行可达性评估、设施选址优化和决策展示打通，直接服务街道与社区在高温来临时的资源调度。”
