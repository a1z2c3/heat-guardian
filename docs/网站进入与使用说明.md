# 网站进入与使用说明

## 1. 先说结论

- 这个项目当前默认是**本地网站**，不需要先买域名。
- 你只要在本机启动服务，然后用浏览器打开 `http://127.0.0.1:8000/` 就能进入网站。
- 对于比赛答辩、老师查看、线下演示，**本地启动就够用**。
- 只有当你想让校外人员、评委、老师通过公网长期访问时，才需要考虑服务器、域名、备案、HTTPS 这些部署问题。

---

## 2. 项目位置

当前项目目录：

```text
D:\bisai
```

当前工程里的核心内容：

- `backend/`：后端接口
- `frontend/`：前端页面
- `data/`：数据文件
- `docs/`：说明文档
- `outputs/`：导出图表和表格

- `生成材料.ps1`：导出报告图表与表格

## 3. 怎么进入这个网站

### 方式一：最推荐，直接一键启动

这是最适合你现在使用的方式。

#### 第一步：打开 PowerShell

可以用以下任一方式：

- 在 `D:\bisai` 文件夹空白处按住 `Shift` 再点鼠标右键，选择“在此处打开 PowerShell”
- 或者先打开 PowerShell，再执行：

```powershell
cd D:\bisai
```

#### 第二步：执行启动脚本

```powershell
.\启动项目.ps1
```

这个脚本会自动做几件事：

1. 如果没有虚拟环境，就自动创建 `.venv`
2. 自动安装 `requirements.txt` 里的依赖
3. 自动执行数据流水线
4. 自动启动 `FastAPI + Uvicorn` 服务

这里的“数据流水线”不是演示脚本，而是真实刷新链路。启动时会先更新 `WorldPop / Geofabrik / Open-Meteo / 武汉官方公开页面`，再把处理后的结果提供给网站读取。

#### 第三步：看到启动成功标志

当终端里出现类似下面的内容时，说明网站已经启动成功：

```text
Uvicorn running on http://127.0.0.1:8000
```

#### 第四步：打开浏览器进入网站

在浏览器地址栏输入：

```text
http://127.0.0.1:8000/
```

就可以进入系统首页。

首页现在新增了“证据链”板块，会直接展示：

- 本轮是否已经自动刷新上游数据
- `WorldPop` 和 `Geofabrik` 最近检查时间
- 官方纳凉通报监测页数与原文入口
- 方法依据和外部参考链接

---

### 方式二：手动启动

如果你不想用脚本，也可以手动执行。

```powershell
cd D:\bisai
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python scripts\run_pipeline.py
uvicorn backend.app.main:app --host 127.0.0.1 --port 8000
```

然后浏览器打开：

```text
http://127.0.0.1:8000/
```

---

### 方式三：让同一局域网里的其他电脑进入

如果你想让同一个宿舍、实验室、办公室里的另一台电脑访问你这台电脑上的网站，可以改成下面这种启动方式：

```powershell
cd D:\bisai
.\.venv\Scripts\Activate.ps1
uvicorn backend.app.main:app --host 0.0.0.0 --port 8000
```

然后其他电脑访问你的局域网 IP，例如：

```text
http://192.168.1.23:8000/
```

注意：

- 这不是公网访问，只是同一网络下访问
- 你的电脑防火墙可能需要放行 `8000` 端口

---

### 方式四：公网访问

这一步**不是现在必须做的**。

只有以下情况才建议考虑：

- 你想给外校老师远程看
- 你想长期放到网上
- 你想做正式线上演示地址

如果做公网访问，通常需要：

1. 一台云服务器
2. 项目部署到服务器
3. 一个域名（可选但推荐）
4. 域名备案（中国大陆常见要求）
5. HTTPS 证书

所以结论是：

- **本地用：不需要域名**
- **答辩演示：通常也不需要域名**
- **正式公网展示：才考虑域名和部署**

## 4. 第一次启动时你会看到什么

第一次启动通常会比后面慢，因为系统可能在做：

- Python 依赖安装
- 天气数据抓取
- POI 数据抓取
- 老年人口数据准备
- 路网可达性计算
- 风险网格生成
- 推荐点位计算

如果电脑没断网，首次等待几分钟是正常的。

后续再次进入，一般只要重新执行：

```powershell
cd D:\bisai
.\启动项目.ps1
```

即可。

## 5. 网站首页怎么使用

当前前端是**单页仪表盘**，打开后从上到下看即可。

### 5.1 顶部总览区

你一进入页面，最上方会看到：

- 研究区
- 最高风险城区
- 完整模型增益
- 路网替代误差
- 决策摘要

这一块的作用是让老师、评委、队友在几秒内知道：

- 研究对象是谁
- 当前最危险区域在哪里
- 你的模型比简化模型强在哪里
- 你的选址策略用了什么逻辑

---

### 5.2 状态提示条

在顶部卡片下面会有状态提示条。

它会告诉你：

- 当前是否已经成功联动真实数据
- 当前展示的是哪个方案
- 最近更新时间

如果加载失败，这里也会显示错误或警告，并提供“重新加载”按钮。

---

### 5.3 指标卡片区

这一排卡片用于快速看核心指标，包括：

- 当前温度
- 24 小时最高温
- 高风险网格数
- 15 分钟覆盖率
- 现有避险资源数量
- 默认方案覆盖率
- 默认方案新增覆盖人数
- 路网替代误差
- 完整模型增益

这部分适合答辩时先讲“问题有多大、模型有多强、方案效果有多明显”。

---

### 5.4 角色行动看板

这一块是给不同角色看的，不只是做展示。

可以切换三个角色：

- `街道调度`
- `社区执行`
- `卫健统筹`

切换后，系统会自动改变：

- 当前视角下的优先级
- 对应指标解释
- 该角色应该采取的行动建议

这块非常适合比赛答辩，因为它体现出：

- 你不是只做了一个算法
- 你是把结果变成了可执行的治理建议

---

### 5.5 方案切换与候选点明细

这一块可以切换不同选址情景：

- `新增 3 点`
- `新增 5 点`
- `新增 8 点`

切换后会同步变化：

- 覆盖率
- 新增覆盖人数
- 平均到达时间
- 时间收益
- 候选点清单

你可以把它理解为：

- `3 点方案`：更节省资源
- `5 点方案`：当前默认推荐
- `8 点方案`：更强但成本更高

这块是你展示“资源投入—收益变化”的关键部分。

---

### 5.6 真实空间态势图

这是当前前端里最适合答辩展示的一块。

它会把以下内容放到一个空间面板里：

- 风险网格
- 当前方案候选点
- 最高风险网格列表
- 当前候选点位列表
- 空间聚焦详情

#### 你可以怎么用：

1. 点击左侧地图里的某个风险网格
2. 右侧会显示该网格的风险分、老年人口、最近步行时间等信息
3. 点击右侧“最高风险网格”列表，也会同步聚焦
4. 点击“当前候选点位”列表，可以看每个新增点位覆盖了多少人、改善了多少网格

这块适合回答两个问题：

- 风险具体发生在什么空间位置
- 推荐点位为什么选在那里

## 8. 怎么关闭网站

当网站正在运行时，终端窗口不要直接关。

正确关闭方式：

- 在运行 `uvicorn` 的 PowerShell 窗口按 `Ctrl + C`

这样服务会正常停止。

## 9. 常见问题

### 9.1 打不开网页

先检查三件事：

1. 是否已经执行了 `.\启动项目.ps1`
2. 终端里是否出现了 `Uvicorn running on http://127.0.0.1:8000`
3. 浏览器地址是否输入正确：

```text
http://127.0.0.1:8000/
```

---

### 9.2 PowerShell 不让运行脚本

如果提示脚本执行被系统限制，可以临时执行：

```powershell
Set-ExecutionPolicy -Scope Process Bypass
```

然后再执行：

```powershell
.\启动项目.ps1
```

---

### 9.3 8000 端口被占用

如果 `8000` 端口被其他程序占用，可以换端口启动，例如：

```powershell
uvicorn backend.app.main:app --host 127.0.0.1 --port 8010
```

然后浏览器访问：

```text
http://127.0.0.1:8010/
```

---

### 9.4 页面打开了但数据不完整

可能原因：

- 数据流水线还没跑完
- 某些外部接口当时网络不稳定
- 风险网格没有成功返回

建议按顺序处理：

1. 执行 `.\更新数据.ps1`
2. 再执行 `.\启动项目.ps1`
3. 页面里点击“重新加载”

---

### 9.5 一定要部署到服务器吗

不一定。

对于你现在这个比赛项目：

- **本地展示可以正常用**
- **答辩演示可以正常用**
- **给老师现场看可以正常用**

只有在你明确需要“别人不在你电脑旁边也能访问”时，才需要服务器部署。

---

### 9.6 一定要买域名吗

也不一定。

结论如下：

- 只在自己电脑上用：**不要域名**
- 同一局域网临时访问：**不要域名**
- 公网长期访问：**建议域名**
